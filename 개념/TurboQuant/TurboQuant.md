**개념**

1. quantization [quantization](./quantization.md)
2. kv cache [kv cache](./kv_cache.md) (효율화 하고자 하는 대상)

**개요** <br>
kv cache의 key를 효율적으로 quantization하는 방법
   - value는요? => value는 Key와 달리 값이 튀는 현상이 심하지 않고, 상대적으로 균일하고 얌전한 분포를 가짐
                    => 이런 방법없이 쉽게 quantization가능

    - 그럼 key가 값이 튀는것 같은데 왜요?
      - 소프트맥스(Softmax)의 특성과 '주목할 곳이 없을 때'의 임시방편
        - Softmax을 진행할때
          - , . / ; : 와 같이 문맥에 크나큰 영향이 없을때 참고하는 일명 "더미 저장소"가 점점 생기게 되는데 이 과정에서 특정 key값이 튈수 있음
        - 레이어 정규화
          - 정규화를 진행하다 보니 중요한 정보가 강제로 잘릴수 있음, 그래서 값을 키워서 지우지 못하게 한다음 해당 정보를 유지하도록 하는 과정에서 특정 key값이 튈수 있음
        - Key와 Value의 역할 차이
          - key는 어디를 볼지 알려주는거고 value는 데이터의 값이기 때문에 key가 확실하게 전달될수 있도록 하는 과정에서 특정 key값이 튈수 있음
     
     - 그럼 쉽게 바로 key만 수정하면 되는거 아닌가요?
       - 튀는값의 규칙이나, 값, 위치가 정해져 있지 않음으로 그냥 잡아서 quantization하기에는 어려움

**방법**<br>
- L2 normalize (범위 줄이기, 튀는값 제거 X)
  - 데이터의 범위가 일정하지 않을때 "성질(비율)"은 유지하면서 특정 범위로 줄이는 기술
    - 전체 크기(L2 Norm) 구하기
      - 각 숫자를 제곱해서 더한 뒤 루트를 씌움
    - 원래 데이터를 그 길이로 나누기
      - 원래 데이터였던 값을 전체 크기로 각각 똑같이 나눔

- RHT (튀는값 제거 O)
  - 데이터중 특정 데이터가 높게 튀어있을때에 그 튀는값을 나머지 데이터에 나누어 튀는값을 제거하는 기술
    - R : 무작위로 부호 바꾸기
      - 데이터의 부호를 무작위로 섞음
    - H : 하다마드 변환으로 섞기
      - 특정 튀는값을 나머지 데이터에 나누어줌
        - ex) [0,0,0,100] => [25,25,25,25]

**좋은점**<br>
- 값이 튀는 데이터를 key데이터를 quantization 하기쉬운 데이터로 압축할수 있음
  - => 4bit연산으로도 값이 꺠지지 않고 거의 완벽하게 유지할수 있음
- matrix
  - 압축한 값을 다시 원래대로 돌릴때 사용되는 기술
  - 보통은 RHT의 역행렬을 사용해서 돌리는데 matrix역행렬로 돌려서 공식을 사용하지 않고 곱하기만 해서 결과를 구할수 있음 (+ 메모리 이득 ^^)

**결과**
- 이런 연산을 통해서 적은 공간과 적은 메모리로 Attention Score를 구할수 있음
  - Attention Score? : 단어끼리의 연관도 점수
