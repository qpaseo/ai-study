**개념**

- quantization (양자화)
- 일정 범위의 숫자를 일정한 규격으로 나누어 구분하는것 (연속적인 실수 값을 더 적은 비트의 정수 값으로 변환)
- 숫자를 아주 정밀하게 표시하면 자세하게 표현해서 얻는 정밀함보다 공간이 차지하는게 더 커짐 (진짜 많은 값을 가지고 있기 때문에 [많은 차원의 백터 값으로 이루어져 있기 때문에])=> 비효율적이고 위험함
- 범위는 보통 n게로 나눈다고 하는데 n이 클수록 더 "정밀해짐"(오차가 적다)
  - n은 2의 k승 (k는 메모리 bit수)으로 결정됨

**한계**

- 범위밖의 숫자는 어떻게 처리하는거?
  - 가장 가까운 수에서 차이가 남
    - ex) 범위가 -11~1 인데 2가 들어오면 1이 가장 가까운 수고 1 차이가 나기에 1이 나옴
- 범위랑 n을 모르는 사람이 결과값을 보면 알수 있는가?
  - 모른다
    - 이유 : 2진수로 구간에 따른 출력이 나오기 떄문에 모른다
      - ex[k=4 라는 가정]) 1번째 구간 안에 있는 수는 0000 2번째 구간 안에 있는 수는 0001 이런식으로 구간에 대한 수는 정혀져 있지만 범위 정보가 없기떄문에 몇번째 인지만 알수있기 떄문이다.
      - 범위 정보를 같이 저장해야한다

**확률 분포의 quantization**
- 일반적인 quantization, 확률분포의 quantization 차이
  - 일반적인 quantization : 범위를 "일정한" 간격으로 분리
  - 확률분포의 quantization : 범위중 특별하게 많이 나오거나 특별하게 많이 분포되거나 나오는 부분에 따라 분리 정도를 다르게 하는것(특별한 부분에 모아져 있다는것)
    - 최적화 방법
      - 범위를 줄인다 : 100~200을 탐지하다가 10~20 만 나오는것을 확인하고 범위를 10~20으로 조정한다.
      - 분포를 좁힌다(특정곳만) : 균등하게 자르다가 특정 한 부분에서 많이 나온다면 그 부분을 오차가 적게 해서 정밀하게 측정해야 함으로 범위를 특정 곳만 좁힌다.
        - ex) 10~20이 많이 나오니깐 다른곳에서는 0.5 차로 구분하다가 많이 나오는 부분은 0.2 차이로 구분해서 차이를 줄이가 위해
